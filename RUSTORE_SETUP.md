# Настройка RuStore Billing

## Шаг 1: Регистрация в RuStore

1. Перейдите в [RuStore Консоль](https://console.rustore.ru/)
2. Зарегистрируйтесь как разработчик
3. Создайте новое приложение

## Шаг 2: Получение Console App ID

1. В консоли RuStore откройте страницу вашего приложения
2. Скопируйте ID из URL: `https://console.rustore.ru/apps/123456`
3. В данном примере ID: `123456`

## Шаг 3: Настройка приложения

В файле `app/src/main/res/values/strings.xml` замените значение:

```xml
<string name="rustore_console_app_id">123456</string>
```

на ваш реальный ID из консоли.

## Шаг 4: Создание подписок

1. В консоли RuStore выберите ваше приложение
2. Перейдите в раздел **"Монетизация" → "Подписки"**
3. Нажмите **"Создать подписку"**

### Подписка "Базовая" (3 приложения)

- **ID продукта**: `kresty_basic_monthly`
- **Название**: Базовая
- **Описание**: До 3 изолированных приложений
- **Период**: 1 месяц
- **Цена**: 89 ₽

### Подписка "Безлимит"

- **ID продукта**: `kresty_unlimited_monthly`
- **Название**: Безлимит
- **Описание**: Неограниченное количество изолированных приложений
- **Период**: 1 месяц
- **Цена**: 479 ₽

## Шаг 5: Deeplink

Deeplink уже настроен в приложении:
- Схема: `kresty://`
- Обработка происходит в `SubscriptionActivity`

## Шаг 6: Тестирование

### Требования для тестирования:

1. **RuStore установлен** на устройстве
2. **Пользователь авторизован** в RuStore
3. **Приложение подписано** тем же ключом, что и загруженное в консоль
4. **Приложение прошло модерацию** (можно не публиковать)

### Проверка подписи:

Тестовая сборка должна быть подписана тем же ключом, что и релизная сборка в консоли.

### Тестовые платежи:

В RuStore нет отдельного тестового режима. Для тестирования:
1. Используйте реальные карты
2. Оформляйте подписки
3. Отменяйте подписки через консоль для возврата средств

## Шаг 7: Публикация

1. Загрузите APK/AAB в консоль
2. Заполните информацию о приложении
3. Дождитесь модерации
4. Опубликуйте приложение

## Устранение неполадок

### Ошибка "RuStore не установлен"

Установите RuStore с [официального сайта](https://rustore.ru/)

### Ошибка "Пользователь не авторизован"

Авторизуйтесь в приложении RuStore

### Ошибка "Приложение не прошло модерацию"

Дождитесь завершения модерации в консоли

### Ошибка "Неверная подпись"

Убедитесь, что тестовая сборка подписана тем же ключом, что и в консоли

## Важные замечания

### Подписки в RuStore

- Подписки автоматически продлеваются
- Пользователь может отменить подписку в RuStore
- Разработчик может отменить подписку в консоли

### Комиссия

RuStore взимает комиссию с каждой покупки. Актуальные тарифы смотрите на [сайте RuStore](https://rustore.ru/).
